# 空间量测 DistanceMeasure 点击第二个点崩溃问题

## 问题描述

在空间量测功能中，点击第一个点正常，点击第二个点时页面崩溃并报错：

```
An error occurred while rendering. Rendering has stopped.
TypeError: CollectionConstructor is not a constructor
    at EntityCluster.getLabel (chunk-xxx.js:113417)
    at LabelVisualizer.update (chunk-xxx.js:133174)
    at DataSourceDisplay.update (chunk-xxx.js:136595)
    at CesiumWidget._onTick (chunk-xxx.js:199357)
```

---

## 环境信息

| 依赖 | 版本 |
|---|---|
| cesium | 1.139.1 |
| vite-plugin-cesium | 1.2.23 |

---

## 根因分析

### 崩溃链路

点击第二个点后，`viewer.entities.add()` 创建了带 `label` 的 Entity：

```
用户点击第二个点
  └── viewer.entities.add({ point: {...}, label: {...} })
        └── Entity 加入 DataSourceDisplay
              └── Cesium 每帧调用 DataSourceDisplay.update()
                    └── LabelVisualizer.update() ← 每帧触发
                          └── EntityCluster.getLabel() ← 每帧触发
                                └── new CollectionConstructor({ scene }) ← 崩溃！
                                      └── CollectionConstructor === LabelCollection
                                            └── 构造函数引用丢失，报 TypeError
```

### 崩溃的本质原因

**`vite-plugin-cesium` 1.2.23 与 Cesium 1.139.1 存在打包兼容性问题**。

Cesium 内部 `EntityCluster.js` 使用了如下工厂函数模式：

```javascript
// Cesium 源码（EntityCluster.js）
function createGetEntity(collectionProperty, CollectionConstructor, ...) {
    return function (entity) {
        // ...
        if (!defined(collection)) {
            collection = this[collectionProperty] = new CollectionConstructor({ scene: this._scene });
            //             ↑ 这里是崩溃点
        }
        // ...
    };
}

EntityCluster.prototype.getLabel = createGetEntity(
    "_labelCollection",
    LabelCollection,   // ← 从 cesium 导入的 LabelCollection 构造函数
    "_unusedLabelIndices",
    "labelIndex"
);
```

在 `vite-plugin-cesium` 打包过程中，`cesium` 被标记为 `external`，使用 `rollup-plugin-external-globals` 将其替换为全局变量 `window.Cesium`。然而 Cesium 内部的类引用（如 `LabelCollection`）在打包后的 `Cesium` 全局对象上**没有被正确导出或引用**，导致 `new CollectionConstructor(...)` 时得到 `undefined`，抛出 `TypeError: CollectionConstructor is not a constructor`。

### 关键代码位置

崩溃**不是 DistanceMeasure 本身的代码错误**，而是 Cesium + vite-plugin-cesium 组合的固有兼容性问题。触发点：

```typescript
// DistanceMeasure.ts —— 触发崩溃的代码
const tempPoint = this.viewer.entities.add({
    position,
    point: { pixelSize: 10, color: Color.YELLOW, ... },
    label: {
        text: new CallbackProperty(() => this.getPointLabel(index), false),
        // ↑ 这个 label 字段会激活 LabelVisualizer，进而触发 EntityCluster.getLabel
    }
})
```

## 修复方案

### 核心思路

**绕开 Entity/EntityCluster 系统，使用 Primitive 图元替代**。

| 对比 | Entity.Label | LabelCollection Primitive |
|---|---|---|
| 渲染管线 | Entity → DataSourceDisplay → Visualizer → EntityCluster | Primitive → Scene 直接渲染 |
| EntityCluster | 会触发 | 不会触发 |
| 崩溃 | 是 | 否 |
| 标注更新方式 | 必须用 Property/CallbackProperty | 直接赋值属性 |

### 修复实现

#### 1. 在构造函数中创建 `LabelCollection`

```typescript
import { LabelCollection, Label } from "cesium"

private labelCollection: LabelCollection
private pointLabelObjects: Label[] = []

constructor(viewer: Viewer, dispatcher: EventDispatcher) {
    super(viewer, dispatcher)
    this.measureUtil = MeasureUtil.getInstance(viewer)

    // 创建 LabelCollection 并加入场景
    this.labelCollection = new LabelCollection({ scene: viewer.scene })
    viewer.scene.primitives.add(this.labelCollection)
}
```

#### 2. 落点时用 `LabelCollection.add()` 而非 `Entity.label`

```typescript
// 用 LabelCollection primitive 添加标注（图元系统，不触发 EntityCluster）
const labelObj = this.labelCollection.add({
    position,                                    // 直接用 Cartesian3
    text: `空间距离：${totalSpaceText}(计算中...)`,
    font: '18px sans-serif',
    fillColor: Color.CHARTREUSE,
    outlineColor: Color.BLACK,
    outlineWidth: 2,
    style: LabelStyle.FILL_AND_OUTLINE,
    verticalOrigin: VerticalOrigin.BOTTOM,
    horizontalOrigin: 0,
    pixelOffset: new Cartesian2(20, -20),
    heightReference: HeightReference.NONE,
    disableDepthTestDistance: Number.POSITIVE_INFINITY,
})
this.pointLabelObjects.push(labelObj)
```

#### 3. 贴地距离计算完成后，直接赋值更新文字

```typescript
// 异步计算完成后，直接更新 Label 对象的 text 属性
// LabelCollection 图元的 text 是普通字符串属性，可直接赋值
void this.measureUtil.calculateGroundDistance(lastPos, position).then((groundResult) => {
    this.totalGround += groundResult.rawValue
    const distanceText =
        `空间距离：${totalSpaceText}(+${spaceResult.formatted})` +
        `\n贴地距离：${totalGroundText}(+${groundResult.formatted})`

    // ✅ 直接赋值字符串，不触发 EntityCluster
    if (this.pointLabelObjects[labelIndex]) {
        this.pointLabelObjects[labelIndex].text = distanceText
    }
})
```

#### 4. 清理时移除 `LabelCollection`

```typescript
private clearLabels(): void {
    if (this.labelCollection) {
        this.labelCollection.removeAll()
    }
    this.pointLabelObjects = []
}

destroy(): void {
    this.clearLabels()
    if (this.labelCollection && this.viewer) {
        this.viewer.scene.primitives.remove(this.labelCollection)
    }
    super.destroy()
}
```

## 其他可能的解决思路

### 方案 A：禁用 EntityCluster（不推荐）

在 CesiumViewer 初始化时关闭 clustering：

```typescript
viewer.dataSources.get(0).clustering.enabled = false
```

缺点：会影响整个场景的聚合功能，范围太大。

### 方案 B：降级 vite-plugin-cesium（可选）

根据 GitHub Issue #60，`vite-plugin-cesium` 1.2.23 存在 regression，1.2.22 相对稳定。但降级可能引入其他兼容性问题。

### 方案 C：升级 / 不使用 vite-plugin-cesium

考虑改用手动配置 `CESIUM_BASE_URL`，不依赖 `vite-plugin-cesium`。

## 调试方法

### 日志系统

项目中已集成 `MeasureLogger.ts`，日志打印到控制台并同时保存到 `localStorage`。

在 `MeasureTool.vue` 组件中有**下载日志**按钮，可以导出日志文件进行分析。

### 关键日志节点

崩溃发生时，日志会显示 `EXIT onLeftClick` 但紧接着出现 `TypeError`，可以区分是代码逻辑问题还是渲染崩溃。
